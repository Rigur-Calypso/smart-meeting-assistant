# Smart Meeting Assistant

This project is a real-time video meeting application featuring a smart AI assistant. The assistant provides live transcription of the meeting and can answer questions about the conversation when prompted.

The application is composed of two main parts:
1.  A **Next.js frontend** for the user-facing meeting interface.
2.  A **Python backend** that runs the AI agent, which joins the meeting to provide assistance.

## Features

-   **Real-time Video Meetings**: Join video calls with multiple participants.
-   **Live Transcription**: Get a real-time transcript of the ongoing conversation.
-   **AI-Powered Q&A**: Ask the assistant questions about the meeting content by saying "Hey Assistant," and get concise answers.
-   **Separate Frontend and Backend**: A modern web stack with a Next.js frontend and a Python-based AI backend.

## Getting Started

### Prerequisites

-   Node.js and npm
-   Python 3.13+
-   Stream account credentials (`API_KEY`, `API_SECRET`)
-   Google Gemini API Key (`GEMINI_API_KEY`)

### Setup

1.  **Clone the repository:**
    ```sh
    git clone <your-repository-url>
    cd <your-repository-folder>
    ```

2.  **Configure Environment Variables:**

    You need to set up environment variables for both the frontend and backend.

    *   **Backend (`/backend/.env`):**
        Create a `.env` file in the `backend` directory and add your credentials. The `CALL_ID` should be a unique identifier for the meeting room.

        ```
        // filepath: backend/.env
        STREAM_API_KEY=YOUR_STREAM_API_KEY
        STREAM_API_SECRET=YOUR_STREAM_API_SECRET
        CALL_ID=YOUR_UNIQUE_CALL_ID
        GEMINI_API_KEY=YOUR_GEMINI_API_KEY
        ```

    *   **Frontend (`/meeting-assistant/.env.local`):**
        Create a `.env.local` file in the `meeting-assistant` directory. Use the same `STREAM_API_KEY` and `CALL_ID` as in the backend configuration.

        ```
        // filepath: meeting-assistant/.env.local
        NEXT_PUBLIC_STREAM_API_KEY=YOUR_STREAM_API_KEY
        NEXT_PUBLIC_CALL_ID=YOUR_UNIQUE_CALL_ID

        # These are used by the Next.js backend route to generate tokens
        STREAM_API_KEY=YOUR_STREAM_API_KEY
        STREAM_API_SECRET=YOUR_STREAM_API_SECRET
        ```

### Installation

1.  **Install Backend Dependencies:**
    Navigate to the `backend` directory and install the Python packages. It's recommended to use a virtual environment.

    ```sh
    cd backend
    python -m venv .venv
    source .venv/bin/activate  # On Windows use `.venv\Scripts\activate`
    pip install -r requirements.txt # Or use `uv pip install -r requirements.txt`
    ```
    *(Note: A `pyproject.toml` is provided, so you can also use tools like `uv` or `pip` to install from it).*

2.  **Install Frontend Dependencies:**
    Navigate to the `meeting-assistant` directory and install the npm packages.

    ```sh
    cd meeting-assistant
    npm install
    ```

### Running the Application

1.  **Start the Backend Agent:**
    Open a terminal, navigate to the `backend` directory, and run the Python script.

    ```sh
    cd backend
    python main.py
    ```
    The AI assistant will start and attempt to join the meeting specified by `CALL_ID`.

2.  **Start the Frontend Application:**
    Open a second terminal, navigate to the `meeting-assistant` directory, and run the development server.

    ```sh
    cd meeting-assistant
    npm run dev
    ```
    Open [http://localhost:3000](http://localhost:3000) in your browser to access the application.

### How to Use

1.  Open the web application at `http://localhost:3000`.
2.  Enter your name and click "Join Meeting".
3.  You will be placed into the video call. The AI assistant should join shortly after.
4.  As participants speak, you will see a live transcript appear on the right-hand panel.
5.  To ask the assistant a question, say "Hey Assistant," followed by your query (e.g., "Hey Assistant, what were the main points discussed?"). The assistant will respond verbally.
